Understanding Git and GitHub
What is Git?

Git is a distributed version control system

  • A version control system is a system that tracks and manages changes to folders/files.
  • A distributed version control system takes that same system and adds a peer-to-peer model to it.

Git is NOT the same as Github. Github is a platform built on top of Git.

Why use Git?
  • Track every change: See the complete history of your project and easily revert to previous versions.
  • Collaborate seamlessly: Work on the same project with others without conflicts or messy merging.
  • Experiment without risk: Use branches to try out new features without affecting the main codebase.
  • Recover from disasters: Never lose your work again with Git's complete version control.

Git Project Flow

  1. Initialize repository
  2. Stage changes
  3. Commit changes
  4. Branching
  5. Merging
  6. Remote repository
  7. Pull changes

Initialize a Repository

  • To create the .git directory

    • Git Directory

    • The .git directory contains all of the git data, from the upstream repository to change history.
    • This is what makes a repository a repository

    Staging Changes

    Staging is the process of adding files to be tracked.

    • To stage changes, use the command git add <filename>
      • Multiple files and directories can be separated by spaces
    • To stage all changes, use the command git add .

    Committing Changes

    Committing is the process of saving those staged changes into an object, which can later be pushed.

    • To commit staged changes, use the command git commit -m "<message>"

    Branching

    Allows multiple “save points” for your work.
    • New features can be created a new branch, which is later merged into the main branch
    • If issues arise on one branch, it does NOT impact any other branch
    • To update the remote repo, push changes (git push)

    Remote Repository

    A remote repository is the version of the project, which is hosted on a network.

    • Remote repositories can be hosted on a variety of platforms, such as Github, Gitlab, Bitbucket, etc.
    • GitHub is the most popular
  • There can be multiple remotes
